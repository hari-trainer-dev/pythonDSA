Time & Space Complexity

Let's start with Time & Space Complexity — the foundation of analyzing algorithms efficiently. This is crucial for interviews and real-world performance understanding.

Big-O Notation

Big-O notation describes the upper bound (worst-case) growth rate of an algorithm's running time or space usage as input size n becomes very large. It ignores constants and lower-order terms — we care about the dominant behavior.

Common time complexities (from best to worst):

  • O(1) — Constant time (e.g., accessing an array element by index)
  • O(log n) — Logarithmic (e.g., binary search)
  • O(n) — Linear (e.g., single loop over n elements)
  • O(n log n) — Linearithmic (e.g., efficient sorting like merge sort, quicksort average)
  • O(n²) — Quadratic (e.g., nested loops, bubble sort)
  • O(2ⁿ) — Exponential (e.g., naive recursive traveling salesman, subset generation without memo)
  • O(n!) — Factorial (e.g., naive permutations generation)

Analyzing Loops

To find time complexity, count how many times basic operations run.

  • Single loop from 1 to n → O(n)
  • Nested loops (outer n times, inner n times) → O(n²)
  • Nested with decreasing inner (e.g., i from 1 to n, j from i to n) → still O(n²) (roughly n(n+1)/2)

Rule of thumb: Drop constants and non-dominant terms.

  • 3n + 5 → O(n)
  • 4n² + 100n + 500 → O(n²)

For recursive algorithms, use the recursion tree method or Master Theorem.

Amortized Analysis

Average time per operation over a sequence (useful for dynamic arrays, hash tables).

Example: ArrayList / Vector resize (double when full).

Most appends: O(1)

Occasional resize: O(n)

But amortized over many appends → O(1) per append.

Space Complexity

Measures extra space used (besides input).

  • O(1) — constant extra space (e.g., swapping two variables)
  • O(n) — linear space (e.g., copy of array, recursion stack depth n)
  • In-place algorithms try for O(1) extra space

Best / Average / Worst Case Analysis

  • Worst case — Big-O (guaranteed upper bound, most important for interviews)
  • Best case — usually Ω (lower bound), e.g., already sorted quicksort → O(n)
  • Average case — expected over random inputs (e.g., quicksort O(n log n))
